## 学到的东西

- v0.4.0 这么大的更新居然没打 tag — 每次发版后必须检查 `git tag -l` 确认 tag 存在
- 动态 badge (`shields.io/pypi/v/dirsort`) 比硬编码 badge 更专业，自动跟随发布版本
- 之前 README 的"English"链接指向自身是个明显的循环引用 bug — 说明每次都需要读一遍结果验证
- 双语 README 是低成本高收益的推广手段，特别是在英文 GitHub 社区
- 每周的"版本更新横幅"让回头客一眼看到变化，是 README 留存策略的一部分
- CI badge 链接需要验证是否真实可用 — 即使 workflow 文件存在，也需要推送到 GitHub 后才能生效
