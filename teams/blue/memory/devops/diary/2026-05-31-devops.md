## 遇到的坑

1. **workflow scope 双杀**: ci.yml 和 publish.yml 都被拒绝，需要两次移除才能推送
2. **editable install**: 新环境首次跑测试需要先 `pip install -e ".[dev]"`
3. **checkout 无法恢复**: `git rm --cached` 后 `git checkout --` 无法恢复，需用 `git add` 重新添加

## 学到的东西

- 每次 v0.x.0 发版前，先检查是否有 workflow 文件需要临时移除
- Release 两步法（最小 body → PATCH）稳定可靠，中文/长 body 不会超时
- 全在线模式（API + git）是最佳情况，直接 git push + git tag 即可
