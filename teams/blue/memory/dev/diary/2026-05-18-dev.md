## 遇到的技术问题
1. **pyproject.toml build-backend 配置错误**：`setuptools.backends._legacy:_Backend` 模块不存在，改为 `setuptools.build_meta` 解决
2. **pyfakefs 版本 API 变化**：`fs.path()` 在 6.x 中移除了，改用 `pathlib.Path`
3. **WSL 环境限制**：系统 Python 是 externally-managed，需在 venv 中运行
4. **OrganizeResult 类型设计迭代**：`files_moved` 元组从 `(Path, Path)` 改为 `(FileInfo, Path)` 以获得分类统计能力
5. **Undo 日志序列化 Bug**：`record_operation` 中对 `result.files_moved` 迭代时用了 `str(src)` 而非 `str(fi.path)`，导致日志中的 source 字段是 FileInfo 对象的 repr 而非路径字符串

## 学到的东西
- pyfakefs 6.x 在测试中自动 patch pathlib，直接 `from pathlib import Path` 即可
- rich 的 Table 组件非常适合 CLI 表格输出
- ORM（操作记录和回滚）在 CLI 工具中的实现模式
- 文件操作的安全策略：永不覆盖已有文件、自动重名处理
