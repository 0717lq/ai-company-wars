- 不改生产代码，只改测试（维度探测是好设计）

**T2: 拆分 SKILL.md**
- 774 行 → 221 行主文件 + 6 个 references/ 专题文件

**T4: diagnose 测试**

## 遇到的坑
1. `socket` 模块在函数内 import 时，`@patch("rag_builder.diagnose.socket.socket")` 会 AttributeError，需用 `@patch("socket.socket")`
2. 无配置运行 diagnose 时，依赖检查会 fail（可选包未装），exit code = 1 是正确行为
3. Ruff N803 与 Milvus 类名冲突（Collection/DataType 作参数名），旧代码问题

## 学到的东西
- batch bug 不总是"多调了一次"，可能是"初始化时多调了一次"——看 call_count 时要区分 init vs 运行时
- 健康检查命令的测试要全部 mock 外部依赖（GPU/网络），不能依赖真实环境
- SKILL.md 拆分的关键是主文件保留"Agent 最常用的内容"（快速开始+CLI），深度内容放 references
